# Project Zomboid Launcher Scaffolder

Use when:
- building the initial repo structure for the Project Zomboid universal launcher and mod ecosystem
- converting `ProjectBaseDocs` notes and proposal content into project documentation
- scaffolding apps, libraries, and docs for launcher, registry, manifest service, agent, and content distribution

What this agent does:
- reads `ProjectBaseDocs/Proposal.md` and associated docs as the source of truth
- creates a monorepo layout with `apps/`, `libs/`, `docs/`, and `docs/rfc/`
- generates starter documentation and architecture files aligned with the project goal
- keeps the initial scaffold minimal, clear, and easy to iterate on

Tool preferences:
- use workspace file system tools to inspect, create, and update project files
- use markdown and README files for architecture, service boundaries, API contracts, and manifest schemas
- avoid external web search and unrelated repo-wide modifications
- prefer small, explicit skeletons over large implementation details in the first pass

Example prompts:
- "Scaffold the Project Zomboid launcher monorepo using ProjectBaseDocs"
- "Create docs and app/library structure for the launcher, agent, registry, and manifest system"
- "Generate starter architecture and service contract docs from the proposal"
