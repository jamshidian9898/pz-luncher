قبل از هر چیز، به نظرم این پروژه **بیشتر شبیه یک Platform Engineering Project است تا یک Launcher**. اگر معماری را از ابتدا درست بچینی، تقریباً می‌توانی توسعه را به Agentهای مختلف بسپاری و هر بخش مستقل جلو برود.

چیزی که من پیشنهاد می‌کنم، یک **Monorepo** است که همه چیز داخل آن قرار دارد.

# سرویس‌های اصلی

## 1. Directory Service

تنها وظیفه:

کشف سرورها.

Database:

```sql
servers

id
name
description
region
tags
manifest_url
heartbeat_at
players
max_players
```

API:

```text
GET /servers

GET /servers/{id}

POST /heartbeat
```

---

## 2. Manifest Service

Version history.

```text
server-a

v91
v92
v93
```

API:

```text
GET /manifest/{server}

GET /manifest/{server}/{version}
```

Rollback:

```text
POST /rollback
```

---

## 3. Registry Service

شبیه Docker Registry.

Database:

```sql

size

created_at
```

```sql

```

Storage:

```text
MinIO

    sha256.zip
```

مسئول:


# Provider System

بزرگ‌ترین برگ برنده معماری.

Interface:

```go
type Provider interface {

    Exists(ctx context.Context, pkg Package) bool

    Download(
        ctx context.Context,
        pkg Package,
        destination string,
    ) error
}
```

---

```go

```

