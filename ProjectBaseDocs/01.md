# Project Proposal: Universal Launcher and Mod Ecosystem for Project Zomboid

## Vision

Build a modern ecosystem for Project Zomboid servers that provides:

* Server discovery
* One-click join
* Automatic mod management
* Versioned server packs
* Profile isolation
* Rollback support
* Smart caching
* Decentralized content distribution

without modifying the game itself.

The launcher acts as a layer above Project Zomboid and allows players to switch between servers without manually managing mods, saves, or configurations.

---

# Architecture

```text
Directory Service
        ↓
Manifest System
        ↓
Content Registry
        ↓
Launcher Client
        ↓
Profile Isolation
        ↓
Project Zomboid
```

---

# Components

## 1. Directory Service

Responsible for:

* Public server list
* Search and filtering
* Regions
* Player counts
* Tags
* Favorites
* Heartbeats

Contains metadata only.

No mods are stored here.

---

## 2. Agent

A lightweight process installed beside the dedicated server.

Responsibilities:

* Generate manifests
* Detect mod changes
* Report player count
* Report server status
* Upload content if required

---

## 3. Manifest System

Every server owns a manifest.

Example:

```json
{
    "gameVersion":"42.8",
    "manifestVersion":91,
    "mods":[]
}
```

Manifest defines:

* Build version
* Required mods
* Checksums
* Dependencies
* Download locations

---

## 4. Launcher

Responsible for:

* Server browser
* Download manager
* Integrity checks
* Resume support
* Profile management
* Launching the game

---

## 5. Profile Isolation

Every server receives an isolated environment.

```text
profiles/

    serverA/
        mods/
        saves/
        cache/

    serverB/
        mods/
        saves/
        cache/
```

No conflicts occur between servers.

---

## 6. Global Cache

Shared content cache.

Duplicate downloads are avoided.

```text
cache/
    sha256/
```

Common mods are reused.

---

## 7. Content Addressable Storage

Files are identified by hash instead of filename.

Benefits:

* Deduplication
* Integrity verification
* Efficient updates

---

## 8. Delta Updates

Only changed blocks are downloaded.

Reduces bandwidth dramatically.

---

## 9. Rollback

Manifest history:

* v90
* v91
* v92

Servers can instantly revert to older versions.

---

## 10. Smart Dependency Resolution

Automatically resolves:

* Required mods
* Load order
* Missing dependencies

---

## 11. Server Packs

Examples:

* Vanilla
* Hardcore RP
* PvP
* 10 Years Later

Switching between packs requires no manual work.

---

## 12. Peer Cache (Future)

Clients may exchange chunks between themselves.

Reduces CDN costs.

---

# Content Sources

The system supports multiple sources simultaneously.

## Steam Workshop

Primary source.

Mods are downloaded from Steam whenever available.

Advantages:

* Zero storage cost
* Automatic updates

Disadvantages:

* No version freeze

---

## Community Registry

Hash-based package repository.

Advantages:

* Versioning
* Rollback
* Deduplication

---

## Self-hosted Storage

Server owners may host their own packs:

* MinIO
* S3
* Cloudflare R2

---

## Hybrid Mode

Preferred architecture.

Priority:

1. Local cache
2. Community Registry
3. Server CDN
4. Steam Workshop

This provides maximum resilience.

# Philosophy

The launcher should not replace Project Zomboid.

It should enhance the multiplayer experience and eliminate manual mod management.

## پاسخ سؤال مهمت:

### آیا می‌توان در صورت نیاز فایل‌ها را از Steam دانلود کرد؟

بله، و به نظرم حتی باید این قابلیت وجود داشته باشد.

من معماری Hybrid را خیلی دوست دارم:

```text
Join Server
      ↓
Need Mod X

Local Cache؟
      ↓ no

Community Registry؟
      ↓ no

Server CDN؟
      ↓ no

Steam Workshop؟
      ↓ yes

Download
```

یعنی Steam می‌شود آخرین fallback.

---

حتی Agent می‌تواند برای هر مود مشخص کند:

```json
{
  "id": "Brita",
  "version": "3.2",
  "sha256": "...",
  "sources": [
      "registry",
      "server",
      "steam"
  ]
}
```

در نتیجه اگر:

* Registry خاموش باشد،
* CDN سرور در دسترس نباشد،

لانچر می‌تواند از Steam استفاده کند.

---

به نظر من جذاب‌ترین بخش این پروژه اصلاً Launcher نیست؛ بلکه طراحی چیزی شبیه **Docker Registry برای مودهای Project Zomboid** است، چون اگر خوب طراحی شود، حتی بازی‌های دیگری مثل ARMA، Minecraft، Valheim و Space Engineers هم می‌توانند بعدها از همان معماری استفاده کنند.
