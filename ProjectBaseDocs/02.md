به نظرم اگر می‌خواهی از LLM Agentها (Claude Code، Codex، Cursor، Gemini CLI و ...) استفاده کنی، مهم‌ترین کار این نیست که سریع شروع به کدنویسی کنی، بلکه باید پروژه را به اسناد کوچک و واضح بشکنی. این مدل‌ها وقتی بهترین عملکرد را دارند که معماری و قراردادها از قبل مشخص باشند.

من اگر قرار بود این پروژه را بسازم، احتمالاً ساختار زیر را انتخاب می‌کردم.

قبل از هر چیز، انتخاب تکنولوژی‌ها:

| بخش                  | پیشنهاد            |
| -------------------- | ------------------ |
| Backend Services     | Go                 |
| Agent                | Go                 |
| Launcher Backend API | Go                 |
| Download Service     | Go                 |
| CLI Tools            | Go                 |
| Database             | PostgreSQL         |
| Cache                | Redis              |
| Object Storage       | MinIO / S3         |
| Frontend             | React + TypeScript |
| Launcher             | Wails + React      |
| Packaging            | Wails              |
| Protocol             | REST + WebSocket   |

دلیل Wails:

* خروجی native
* حجم کمتر از Electron
* Go backend داخلی
* Windows + Linux + macOS
* یادگیری Go بیشتر
* مصرف RAM پایین

# سرویس‌ها

## Directory Service

مسئول:

* لیست سرورها
* سرچ
* تگ‌ها
* منطقه
* heartbeat

API:

```text
GET /servers

GET /servers/{id}

POST /heartbeat
```

Database:

```sql
servers
server_tags
server_status
```

---

## Agent

روی Dedicated Server نصب می‌شود.

مسئول:

* مانیتور سرور
* تولید Manifest
* heartbeat
* sync مودها

APIهای مصرفی:

```text
POST /heartbeat

PUT /manifest
```

---

## Manifest Service

نگهداری:

```json
{
  "gameVersion": "42.8",
  "manifestVersion": 91
}
```

تاریخچه:

```text
v89
v90
v91
```

Rollback:

```text
GET /manifest/{server}/{version}
```

---

## Content Registry

شبیه Docker Registry.

نگهداری:

```text
sha256
version
size
metadata
```

API:

```text
POST /packages

GET /packages/{hash}

GET /packages/{hash}/download
```

## Launcher Core

مهم‌ترین بخش پروژه.

این بخش نباید UI داشته باشد.

تنها منطق:

```text
Join Server
↓
Get Manifest
↓
Resolve Dependencies
↓
Check Cache
↓
Download Missing
↓
Prepare Profile
↓
Launch Game
```

پیشنهاد:

Go package مستقل.

بعداً حتی CLI هم می‌تواند از آن استفاده کند.
